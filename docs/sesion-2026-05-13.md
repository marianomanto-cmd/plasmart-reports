### 2. Disparar el deploy en Vercel

Hoy se llegó al límite **free** de Vercel (100 deployments/día). El
merge del PR #10 NO disparó deploy. El sitio sigue corriendo con
`5db2eaa` (versión anterior al feature).

**Mañana (cuando se libere la cuota):**

Opción A — manual y rápida:
- Vercel → Project → Deployments → último deploy de production
  (`5db2eaa`) → menú `⋯` → **Redeploy**.
- Desmarcar "Use existing Build Cache".
- Vercel buildea el HEAD actual de `main` = `60beb63` (que incluye
  este commit de docs además, pero da igual).

Opción B — automática:
- Hacer un push cualquiera a `main` (incluso este mismo commit de
  docs si todavía no se pushea, ver abajo). Vercel detecta el webhook
  y deploya solo.

### 3. Configurar `Ignored Build Step` (opcional, pendiente)

Para no quemar la cuota free con docs/skills:

`Vercel → Project → Settings → Git → Ignored Build Step`:
```
git diff --quiet HEAD^ HEAD ':(exclude).claude/' ':(exclude)*.md'
```
Si el diff entre el commit y su padre es vacío excluyendo `.claude/` y
`.md`, Vercel saltea el build. Útil para commits sólo de docs o skills.
